Security Policy

1. Secure Authentication:

• All users must enter their RM when creating an account in the application to ensure access only for authorized students and personnel.

2. Access Management:

• Establish access levels for different users, ensuring that only authorized individuals have permission to view or modify specific information.

3. Recorded Communication:

• Implement session IDs to serve as a protocol and ensure the security of chatbot conversations, tracking and managing individual interactions between users and the system.

4. Strong Password Policy:

• Require strong passwords and regular changes to prevent unauthorized access. Encourage the use of two-factor authentication for additional security layers.

5. Software Updates and Patches:

• Keep the application and all its components up to date with the latest security fixes to mitigate known vulnerabilities.br>

6. Data Privacy:

• Strict commitment to the privacy of student data, ensuring compliance with data protection regulations and obtaining explicit consent when necessary.

7. Location Permission Restriction:

• Ensure that access to the neighborhood map is restricted only to authorized users, minimizing the risk of location information being misused.
