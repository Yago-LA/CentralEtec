Politica de segurança

1. Autenticação Segura:

• Todos os usuários devem colocar o RM ao criar uma conta no aplicativo para garantir o acesso apenas a alunos e pessoal autorizado.

2. Gerenciamento de Acessos:

• Estabelecer níveis de acesso para diferentes usuários, garantindo que apenas as pessoas autorizadas tenham permissão para visualizar ou modificar informações específicas.

3. Comunicação registrada:

• Implementar id de sessão para servir como protocolo e garantir a segurança das conversas no chatbot, para rastrear e gerenciar interações individuais en-tre os usuários e o sistema.

4. Política de Senhas Fortes:

• Exigir senhas fortes e regulares alterações para evitar acessos não autoriza-dos. Incentivar o uso de autenticação de dois fatores para camadas adicionais de segurança.

5. Atualizações de Software e Patchs:

• Manter o aplicativo e todos os seus componentes atualizados com as últimas correções de segurança para mitigar vulnerabilidades conhecidas.

6. Privacidade dos Dados:

• Compromisso estrito com a privacidade dos dados dos alunos, garantindo conformidade com regulamentações de proteção de dados e obtendo con-sentimento explícito quando necessário.

7. Restrição de Permissões de Localização:

• Garantir que o acesso ao mapa do bairro seja restrito apenas aos usuários autorizados, minimizando o risco de informações de localização serem mal utilizadas.
