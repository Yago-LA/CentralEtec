Política de Seguridad

1. Autenticación Segura:

• Todos los usuarios deben ingresar su RM al crear una cuenta en la aplicación para garantizar el acceso exclusivo a estudiantes y personal autorizado.

2. Gestión de Accesos:

• Establecer niveles de acceso para diferentes usuarios, asegurando que solo personas autorizadas tengan permiso para ver o modificar información específica.

3. Comunicación Registrada:

• Implementar IDs de sesión para servir como protocolo y garantizar la seguridad de las conversaciones en el chatbot, permitiendo rastrear y gestionar interacciones individuales entre los usuarios y el sistema.

4. Política de Contraseñas Fuertes:

• Exigir contraseñas robustas y cambios regulares para prevenir accesos no autorizados. Incentivar el uso de autenticación de dos factores para capas adicionales de seguridad.

5. Actualizaciones de Software y Parches:

• Mantener la aplicación y todos sus componentes actualizados con las últimas correcciones de seguridad para mitigar vulnerabilidades conocidas.

6. Privacidad de los Datos:

• Compromiso estricto con la privacidad de los datos de los estudiantes, garantizando el cumplimiento de las regulaciones de protección de datos y obteniendo consentimiento explícito cuando sea necesario.

7. Restricción de Permisos de Ubicación:

• Asegurar que el acceso al mapa del barrio esté restringido solo a usuarios autorizados, minimizando el riesgo de que la información de ubicación sea mal utilizada.
